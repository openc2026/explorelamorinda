## Notes

- **Footer-template defects across a cluster are a worthwhile catch.** The mojibake + duplicate-town pair likely originated from a single batched edit that introduced the footers — they have the identical structure across all five files, and the defects propagated identically. That's a pattern worth watching: when one page's footer reveals a defect, the entire cluster that shares the footer template is almost certainly affected. Worth a one-pass scan of *other* clusters' footers (compare/, moving-to-lamorinda/, real-estate/, schools/, neighborhoods/) for the same `U+FFFD` defect on the next freshness pass.
- **Replacement-character bugs are silent on Windows consoles.** PowerShell's default console codepage tends to display U+FFFD as a literal `?` or `-`, which makes it easy to miss when scanning files in a terminal. Confirmed the actual code point both before (was U+FFFD on all 15 lines) and after the fix (now U+2014 em-dash) by reading the bytes directly via `[System.IO.File]::ReadAllText` and inspecting the integer character codes. Recommended approach for future passes: when reviewing footer or list-glyph rendering, dump char codes rather than relying on what the console shows.
