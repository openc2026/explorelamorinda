### Infrastructure

- **`scripts/generate-programmatic-pages.js`** — Node generator that reads the YAML manifests and emits Hugo-friendly `_index.md` files with frontmatter and templated bodies. Wrote 13 pages on first run.

## Decisions made

- **Generator over hand-authored pages.** Both relocation and comparison pages are emitted by a single Node script reading YAML. Updates to phrasing or template structure now happen in one place; per-city/per-comparison data sits in YAML where Andrew (or future me) can edit without touching code.
- **Zero-dependency YAML loader.** Wrote a ~40-line YAML parser specifically for our manifest shape rather than adding `js-yaml`. Keeps `package.json` deps minimal; trade-off is the parser only handles the exact indentation we use.
- **Kept existing neighborhood URLs.** Did not duplicate the 12 neighborhood pages into a flat `/neighborhoods/<slug>/` namespace. Instead added an aggregator at `/neighborhoods/` and the canonical YAML manifest, and additively enhanced the existing pages so they pick up FAQPage schema.
- **Soft CTA pattern unchanged.** Used the existing `real-estate-cta-box` HTML pattern (matches every existing real-estate-related CTA on the site) plus the auto-firing `content-cta-banner.html` partial. No new partial files needed.
- **FAQ frontmatter is the single source of truth.** Wired Day-1's FAQPage schema partial to every new page (and the existing 12 neighborhood pages) through frontmatter, not duplicated JSON. Add a `faq:` block to any page and schema lights up automatically.
